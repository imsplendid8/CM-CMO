# 카드뉴스 요약 (한화손보 장기CM)

여러 헤드라인/기사를 **읽기 쉬운 카드 단위**로 압축한다. 분석 톤·상품 비중은 `cm-news-analysis` 스킬과 동일하게 따른다(인사말 금지·시사점 중심·메인 3종 강조).

## 카드 1장 구조

각 카드는 **3줄**을 넘기지 않는다.

- **제목** (≤ 24자): 명사형·임팩트. 낚시성 금지, 사실 기반.
- **요지** (≤ 45자): 무슨 일이고 왜 중요한지 한 줄.
- **담당자 액션** (≤ 30자, 선택): CM 채널이 취할 키워드·메시지·대응.

출처·날짜는 카드 하단에 `출처 · YYYY-MM-DD`로 작게.

## 출력 형식 (택1, 요청에 맞춰)

**(a) 사람이 읽는 카드 목록**
```
① [제목]
   요지 …
   ▸ 액션: …
   출처 · 2026-07-22
```

**(b) news-tool.html 반영용 JSON** (items 스키마 그대로)
```json
{ "t": "제목", "src": "출처", "date": "2026-07-22",
  "url": "https://…", "gist": "요지(≤45자)" }
```
→ `NEWSMON[상품키].items` 배열에 넣는다. `url`은 원문 링크(공개 뉴스만). 제목·요지는 `esc()` 대상이므로 특수문자 그대로 두면 됨.

## 규칙

- 카드당 **1이슈**. 여러 이슈를 한 카드에 섞지 않는다.
- **메인 3종(운전자·주택화재·골프)** 이슈는 앞 순서·더 많은 카드로.
- **수요 트리거**(대형화재·급발진·홀인원·항공결항 등)에 해당하면 요지에 트리거성을 드러내고, 액션에 감시 키워드를 넣는다.
- 광고·홍보성 원문은 카드로 만들지 않는다(사실 이슈만).
- 데이터 거버넌스: 공개 뉴스만, 개인정보·영업비밀 금지, 광고 문구는 광고심의 전 초안.

## 작업 흐름

1. 입력 헤드라인/기사 묶음에서 **중복·광고 제거** → 이슈 단위로 그룹핑.
2. 상품 카테고리 매핑(운전자/주택화재/골프/…, 없으면 업계·경쟁사).
3. 카드당 제목·요지·액션 작성(글자 수 준수).
4. 요청이 대시보드 반영이면 (b) JSON으로, 보고/공유면 (a) 목록으로 출력.
5. 메인 3종 우선 정렬 후 반환.

## 예시

```
① 대형화재 늘며 화재보험 재조명
   화재 건수는 줄지만 피해액 급증 → 주택화재보험 수요 부각
   ▸ 액션: '대형화재·누수 보장' 검색광고 선제
   파이낸셜인사이트 · 2026-07-22
```
