# 보험 약관 용어 순화 (한화손보 CM)

표준약관의 어려운 용어를 **소비자가 이해하는 표현**으로 바꾼다. 목적은 가입 전환율 제고 — 정확성을 지키되 쉽게.

## 3원칙

1. **정확성 우선**: 면책·감액·비례보상 등 불리한 조건은 **오인 없이** 정확히. 쉽게 쓰되 왜곡·과장 금지(광고심의 대상).
2. **소비자 언어**: 한자어·법률어 → 일상어. "담보→보장", "피보험자→보험이 지켜주는 대상", "자기부담금→본인부담".
3. **행동 유도**: 표현이 곧 카피가 되도록 — 강점이면 소구, 불리하면 안심 문구로 프레이밍.

## 용어 변환 출력 형식

각 용어는 아래 4요소로 정리(terms-tool.html의 `TERMS` 스키마와 동일).

```json
{ "t":"용어", "cat":"분류(가입·계약|보장·담보|보상·지급|대상·주체|자동차·운전자|주택화재)",
  "grade":"hard|mid|easy",
  "official":"표준약관 정의(정확·간결)",
  "plain":"소비자 쉬운 표현(가입화면/카피 권장, 1~2문장)",
  "tip":"마케팅 활용 팁(어디에 어떻게 쓸지)" }
```

- `grade`: hard(매우 어려움·최우선 순화)·mid(어려움)·easy(비교적 쉬움).
- `plain`: 존댓말·간결. 불리 조건은 "~할 수 있어요"로 사실 고지하되 불안 최소화.
- 실제 배포 카피는 **광고심의 검수 전제** — 단정·최상급(‘무조건’·‘100%’) 지양.

## 카피 작성 시

- **강점 담보**(즉시개시·정찰제·중복수령 가능 등) → 전면 소구 문구로.
- **불리 조건**(면책·감액·자기부담) → 숨기지 말고 **명확·안심 프레이밍**("이럴 땐 보장이 안 돼요"를 먼저 알리는 게 신뢰·민원↓).
- 상품 라벨: 다이렉트=`한화손보다이렉트 [상품]`, 캐롯=`한화손보 캐롯 [상품]`.

## 현재 작업물 연계

- **`terms-tool.html`** — 약관 용어→소비자 표현 변환 콘솔. `TERMS` 배열(용어별 official/plain/tip/grade/cat)을 위 스키마로 추가·갱신하고 검증·커밋한다.
- **`seo-audit.html`** — 상품별 title/meta/H1·상세 카피에 순화 표현을 반영(광고심의 전 초안).
- **데이터 거버넌스**: 공개 표준약관·샘플 기준. 실제 상품 약관·개인정보·영업비밀 입력 금지. 배포 문구는 광고심의 통과 후.

## 작업 흐름

1. 대상 용어/문구 확정(요청 없으면 grade=hard 용어 우선).
2. 표준약관 정의 확인 → `official` 간결화 → `plain` 순화 → `tip` 작성.
3. terms-tool 반영이면 `TERMS`에 JSON 추가, 카피 요청이면 화면 위치별 문구 제시.
4. 불리 조건 포함 시 정확성·심의 리스크 점검 후 반환.
